# 정보처리기사 실기 핵심 암기 노트

> 2022년 3회 ~ 2026년 1회 기출 11회분 기준 + 2020년 1회 ~ 2022년 2회 이론·SQL 선별(9절). ★ = 2회 이상 반복 출제.
> 실기는 **직접 써야** 하므로, 용어는 한글·영문 표기 둘 다 정확히 외울 것.

## 1. 소프트웨어 설계 (UML · 디자인 패턴)

### UML 구성 ★
- UML = **사물(Thing) + 관계(Relationship) + 다이어그램(Diagram)**
- **클래스**: 속성과 동작(오퍼레이션)을 가진 사물, 직사각형으로 표현
- **인터페이스**: 클래스·컴포넌트의 동작을 모아놓은 것, 단독 사용 불가

### UML 관계 ★ (기호로 구분하는 문제 빈출)
| 관계 | 기호 | 의미 |
|------|------|------|
| 연관(Association) | 실선 | 2개 이상 사물이 서로 관련 |
| 집합(Aggregation) | 속 빈 마름모 ◇ | 부분-전체 (차 ◇— 엔진·바퀴) |
| 포함(Composition) | 속 찬 마름모 ◆ | 전체가 없어지면 부분도 소멸 |
| 일반화(Generalization) | 속 빈 삼각형 △ | 상속, is-a (차 △— 버스·택시) |
| 의존(Dependency) | 점선 화살표 ┄> | 짧은 시간 동안만 관련 (TV ┄> 리모콘) |
| 실체화(Realization) | 점선 + 속 빈 삼각형 | 인터페이스 구현 |

### 다이어그램
- **패키지 다이어그램** ★★: 정적 모델링. 관련 요소(유스케이스·클래스)를 그룹화해 상위 개념으로 추상화, 《import》 등 의존 관계 표현, 시스템 복잡도 감소 (2023-3, 2025-3 동일 문제)
- 구조(정적): 클래스, 객체, 컴포넌트, 배치, 패키지, 복합체 구조
- 행위(동적): 유스케이스, 시퀀스, 상태, 활동, 커뮤니케이션, 타이밍

### 디자인 패턴 ★★★ (거의 매회 출제)
**분류부터 암기: 생성 5개 / 구조 7개 / 행위 11개**
- 생성: Abstract Factory, Builder, Factory Method, Prototype, **Singleton**
- 구조: **Adapter**, **Bridge**, Composite, Decorator, Facade, Flyweight, **Proxy**
- 행위: Chain of Responsibility, Command, Interpreter, **Iterator**, Mediator, Memento, **Observer**, State, Strategy, Template Method, **Visitor**

빈출 정의:
| 패턴 | 키워드 |
|------|--------|
| Singleton ★ | 객체 하나만 생성, 어디서든 참조, 동시 참조 불가, 메모리 낭비 최소화 |
| Proxy ★★ | **대리자**, 내부 복잡한 관계 정리·외부에 세부 내용 은닉 (구조) |
| Adapter ★ | 호환성 없는 인터페이스를 변환해 다른 클래스가 이용 가능하게 |
| Bridge | 구현부에서 **추상층을 분리**, 기능과 구현을 별도 클래스로 |
| Iterator ★ | 접근 잦은 객체에 동일 인터페이스, 내부 노출 없이 **순차 접근** |
| Observer ★ | 한 객체 상태 변화 시 다른 객체들에 전달, **일대다** 의존, 발행/구독 |
| Abstract Factory | 인터페이스로 연관 객체 그룹 생성, **키트(Kit) 패턴** |
| Visitor | 데이터 구조에서 처리 기능 분리, 클래스 수정 없이 연산 추가 |
| 행위 패턴 정의 ★ | 상호작용 방법·책임 분배 정의, 결합도 최소화 |

### 결합도 · 응집도 ★★★ (거의 매회 출제)
**결합도(낮을수록 좋음): 자료 < 스탬프 < 제어 < 외부 < 공통 < 내용**
| 결합도 | 키워드 |
|--------|--------|
| 자료(Data) | 단순 파라미터 전달 |
| 스탬프(Stamp) ★ | **배열·레코드 등 자료 구조** 전달 |
| 제어(Control) ★ | 제어 신호·요소 전달, **권리 전도 현상** |
| 외부(External) | 외부에서 선언된 데이터 참조 |
| 공통(Common) ★ | **전역 변수**(공통 데이터 영역) 공유 |
| 내용(Content) ★ | 다른 모듈의 **내부 기능·자료를 직접 참조/수정** |

**응집도(높을수록 좋음): 기능 > 순차 > 교환(통신) > 절차 > 시간 > 논리 > 우연** ★
- **기능적** ★: 모든 기능이 **단일한 목적**을 위해 수행 (2026-1)
- **순차적** ★: 한 활동의 **출력이 다음 활동의 입력**으로 사용
- **교환적(통신적)** ★: **동일한 입출력**을 사용하는 서로 다른 기능들이 모임 (2026-1)
- **절차적** ★: 다수의 관련 기능을 **순차적으로 수행**하되 데이터 연결은 없음 (2026-1) — 순차적과 구분 주의!
- 외우기: 기·순·교·절·시·논·우

### 요구사항 (2026-1 신규)
- 기능적 요구사항: 시스템이 **무엇을** 해야 하는가 (기능, 입출력)
- **비기능적 요구사항 유형**: **운영**(로그 관리·모니터링), **자원**(메모리 확보, 자원 사용량 제한), **성능**(응답 시간), 신뢰성, 가용성, 유지보수성, 이식성, 보안 등

## 2. 데이터베이스

### 릴레이션 용어 ★★★ (2023-1, 2025-3 동일 문제)
| 용어 | 의미 |
|------|------|
| 튜플(Tuple) | 행(Row), 파일의 **레코드**에 해당 |
| 속성(Attribute) ★ | 열(Column), 데이터베이스의 **가장 작은 논리적 단위**, 개체의 특성 기술 |
| 카디널리티(Cardinality) ★★ | **튜플의 개수** |
| 디그리/차수(Degree) ★★ | **속성의 개수** |
| 도메인(Domain) ★ | 한 속성이 취할 수 있는 **같은 타입 원자값들의 집합** |
| 릴레이션 스키마 | 릴레이션의 구조(틀) |
| 릴레이션 인스턴스 ★ | 실제 값을 가진 튜플들의 집합 |

### 키(Key) ★
- **후보키**: 유일성 + 최소성 모두 만족
- **슈퍼키**: 유일성 O, 최소성 X
- **기본키**: 후보키 중 선택된 키, NULL·중복 불가
- **대체키**: 기본키를 제외한 나머지 후보키
- **외래키** ★: 다른 릴레이션의 기본키를 참조하는 속성

### 무결성 ★★
| 무결성 | 내용 |
|--------|------|
| 개체 무결성 ★ | **기본키**는 NULL·중복 불가 (제약 대상: 튜플, 릴레이션당 1개) |
| 참조 무결성 ★★ | **외래키**는 NULL이거나 참조 릴레이션의 기본키 값과 동일 |
| 도메인 무결성 | **속성** 값은 정의된 도메인에 속한 값 (제약 개수 = 속성 개수) |

### 스키마 3계층 ★
- **개념 스키마**: 전체적인 논리적 구조, 조직 전체 DB (1개)
- **내부 스키마**: 물리적 저장 구조 (레코드 형식, 물리적 순서)
- **외부 스키마**: 사용자·응용 프로그래머 관점의 논리적 구조 (서브 스키마)

### 데이터베이스 설계 순서 ★★ (2023-2, 2026-1 재출제)
**요구 조건(요구사항) 분석 → 개념적 설계(E-R) → 논리적 설계(테이블 변환) → 물리적 설계(저장 구조) → 구현**

### 정규화 ★
- 외우기: **도부이결다조** (1NF 도메인 원자값 → 2NF 부분 함수 종속 제거 → 3NF 이행 종속 제거 → BCNF 결정자가 후보키 → 4NF 다치 종속 제거 → 5NF 조인 종속 제거)
- 이행적 함수 종속(A→B, B→C일 때 A→C) 제거 = **제3정규형** ★
- **반정규화(Denormalization)** ★: 성능 향상 위해 의도적으로 통합·중복·분리(정규화 위배). 일관성·정합성 저하 위험

### 관계대수 ★★ (기호 문제 빈출)
| 연산 | 기호 | 비고 |
|------|------|------|
| Select | **σ** | 튜플(행) 추출, 수평 연산 |
| Project | **π** | 속성(열) 추출, 수직 연산, 중복 제거 |
| Join | **⋈** | 공통 속성으로 두 릴레이션 결합 |
| Division | **÷** | S의 속성값을 모두 가진 튜플에서 S 속성 제외 |
| 합집합 ∪ / 차집합 − / 교차곱 × | | |

- R ÷ S: S의 모든 값과 짝을 이루는 값만 남김 ★
- 조인 종류 ★: **세타 조인**(비교 연산자) → **동등 조인**(=) → **자연 조인**(중복 속성 제거)

### SQL 핵심 ★★★
- `INSERT INTO 테이블 VALUES (...)` / `INSERT INTO 테이블 (속성...) SELECT ... FROM ...`
- `UPDATE 테이블 SET 속성=값 WHERE ...` / `DELETE FROM 테이블 WHERE ...`
- `DROP VIEW 뷰명 CASCADE` ★: 참조하는 데이터까지 연쇄 제거 (↔ RESTRICT)
- `ON DELETE CASCADE` ★: 부모 행 삭제 시 참조하는 자식 행도 삭제
- **COUNT(속성)은 NULL 제외**, COUNT(*)는 NULL 포함 ★★
- DISTINCT: 중복 제거 ★ / GROUP BY + HAVING(그룹 조건, 집계함수 사용) ★
- LIKE: `S%`(S로 시작), `%T%`(T 포함) / IN (a, b) = OR 연결
- UNION: 중복 제거 합집합 (UNION ALL은 중복 유지) ★
- AND가 OR보다 우선순위 높음 ★
- CROSS JOIN: 교차곱 (m×n행)
- **외래키 제약 구문** (2026-1): `CONSTRAINT 제약명 FOREIGN KEY (열) REFERENCES 참조테이블(참조열)`

### 기타
- **파일 구조**: 순차 / **색인(Index)** / 해싱 — 색인은 〈값, 주소〉 쌍 (2025-2)
- **E-R 다이어그램** ★: 사각형=개체, 마름모=관계, 타원=속성, 선=연결, 밑줄=키

## 3. 프로그래밍 언어 함정 포인트 (코드 문제 절반 이상)

### Java 빈출 함정 ★★★
- **오버라이딩된 메서드는 동적 바인딩**(실제 객체 기준), **필드와 static 메서드는 참조 타입 기준** (2024-3 #14: 21+3+21+7=52, 2025-2 #18: "5P")
- `==`는 주소 비교, `equals()`는 내용 비교. 문자열 리터럴은 String Pool 공유 → `"A"=="A"` true, `new String("A")`는 별도 객체 (2023-2 #14: true/false/true/true)
- 생성자 호출 순서: 자식 생성자 → (super) 부모 생성자 먼저 실행 → 부모 생성자 안에서 호출한 메서드도 **오버라이딩된 자식 메서드** 실행 (2025-1 #11)
- `this(값)`: 같은 클래스의 다른 생성자 호출 (2023-1 #20: 5000)
- static 변수는 클래스 공유 → 싱글톤 카운트 문제 (2024-1 #5: 4)
- static 메서드에서 인스턴스 변수 참조 불가 → 컴파일 에러 (2023-3 #12)
- 제네릭 T는 컴파일 시 Object 취급 → 오버로딩 시 print(Object) 호출 (2024-3 #18: "B0")
- 예외: 구체적 catch부터 순서대로, **finally는 항상 실행** (2024-3: 101, 2025-1: 출력1출력5)
- 배열은 참조 전달 → 메서드에서 원소 변경 시 원본 반영, 참조변수 재할당은 미반영 (2025-2 #4: "BB")
- 오버로딩: `calc(String)` vs `calc(int)` — 인수 타입으로 구분 (2025-1 #16)
- enum: `values()[i]`, `name()`, `ordinal()` (2025-3 #6)
- 인터페이스 구현 = `implements`, 상속 = `extends`, 부모 생성자 = `super(...)`
- **오버로딩은 컴파일 타임 결정**: 부모 클래스 안에서 호출하면 부모에 보이는 시그니처로 고정, 실행 시 오버라이딩된 버전만 동적 바인딩 (2026-1: f(Object) 선택 → 자식의 f(Object) 실행, 자식의 f(String) 오버로딩은 무시)
- `int + int + "문자열"`은 왼쪽부터: 9+2+"2"+"3" → "1123" (2026-1)

### C 언어 빈출 함정 ★★★
- 포인터 연산: `p->g + n`은 문자열 시작을 n칸 이동 (2025-3 #5)
- `char *s = "ABC"` / 배열명 = 첫 주소, `*(p+i)` = `p[i]`
- 구조체 연결 리스트: next 재연결 순서 따라가기 (매회 출제 수준)
- static 지역 변수: 값 유지 (2024-3 #6: 20)
- swap(값 전달)은 원본 불변 ★
- switch: **break 없으면 낙하(fall-through)**, `deafult:` 같은 오타는 default가 아니라 레이블 (2024-2 #5: -13)
- 비트 연산: `&`(AND), `^`(XOR), `<<`/`>>`(시프트 = ×2/÷2), 연산자 우선순위 `>>`가 `&`보다 높음 (2025-3 #17)
- 이중 포인터: `(*pptr)[i]`, `*(*arr+i)` 형태 (2025-1 #12, 2024-3 #19)
- **함수 포인터** 구조체 멤버: `int* (*fn)(int*)` — `mine.fn(n)`으로 호출 (2026-1)
- `%x`는 16진수 출력: 32 → "20" (2026-1) / 배열 매개변수 `int p[]`와 `int *p`는 동일
- 배열 순위 구하기, 완전수(6, 28), 소수, 팩토리얼(7!=5040) 패턴

### Python 빈출 함정 ★★
- 슬라이싱: `a[:3]`, `a[4:6]`, `a[29:]`, `a[::2]`(짝수 인덱스), `a[1::2]`, `a[-i-1]`(뒤에서), `a[::-1]`(역순), `a[::-2]`(역순 2칸씩: [0..9] → 9,7,5,3,1) (2026-1)
- **얕은 복사 함정**: `b = m[:]`는 바깥 리스트만 복사, **내부 리스트는 공유** → `b[i] += ...`는 제자리 확장이라 m도 변경됨 (2026-1)
- set: 중복 없음·순서 없음, `add/remove/update`, `&`(교집합) (2023-1 #15, 2025-2 #17)
- dict: `{k: v for ...}`, `values()`, enumerate (2025-3 #11)
- `type(x) == type(100)`: int/str/float/tuple 구분 (2024-3 #12)
- `map(lambda ...)`, `split()`, f-string
- 리스트는 참조 전달 → 함수 안 스왑이 원본에 반영 (2024-3 #2)

## 4. 운영체제

### 스케줄링 ★★★
| 기법 | 방식 | 키워드 |
|------|------|--------|
| FCFS | 비선점 | 먼저 도착한 순서 |
| **SJF** ★★ | 비선점 | 실행 시간 최단 프로세스 우선, 평균 대기 최소, **긴 작업 무한 연기** |
| **SRT** ★★ | 선점 | 남은 시간 최단 우선, 실행 시간 추적 **오버헤드** |
| **RR** ★★ | 선점 | 시분할, Time Slice 소진 시 큐 뒤로, 할당량 작으면 문맥 교환 증가 |
| HRN | 비선점 | (대기시간+실행시간)/실행시간 큰 것 우선 |

**평균 대기 시간 계산법** ★: 대기시간 = 완료시각 − 도착시간 − 실행시간
- RR(quantum 4, A0/8 B1/4 C2/9 D3/5) → 11.75초 (2025-2)
- SRT(같은 데이터) → 6.5초 (2024-2)

### 페이지 교체 ★★
- **LRU**: 가장 오랫동안 사용 안 된 페이지 교체
- **LFU**: 사용 빈도 가장 낮은 페이지 교체
- 계산 문제: 프레임 표 그려서 페이지 부재(Fault) 수 세기 (LRU 12회 문제 = OS 교과서 고전)

### UNIX/Linux ★★
- 명령어: `pwd`(현재 경로), `ls`(목록), `cd`(이동), `cp`(복사), `cat`, `chmod`, `fork`, `ps` (2025-3)
- **chmod 8진수** ★: r=4, w=2, x=1 → rwxr-x--x = **751** (2023-3)

### 프로토콜 ★★★
| 용어 | 핵심 |
|------|------|
| **ARP / RARP** ★ | IP → MAC 변환 / MAC → IP 변환 |

| **RIP** ★★ | 거리 벡터, **Bellman-Ford**, 최대 15홉(홉 수 기준 경로 선택), 소규모 |

| **ATM** | 53Byte 고정 셀, B-ISDN, CBR/VBR |
| **NAT** | 공인 IP ↔ 사설 IP 변환, IP 마스커레이드 |
| **L2TP** | PPTP + L2F 결합, 2계층 터널링, 자체 암호화 없음 |
| **IPSec** ★ | 네트워크 계층 IP 패킷 보호, AH·ESP·SA·IKE, VPN에 사용 |
| **SSH** ★★ | 원격 로그인·명령·파일 복사, 공개키 인증, **22번 포트** (2023-1, 2025-2 동일) |
| **AJAX** ★★ | 자바스크립트+XML 비동기, 페이지 일부만 갱신 (2023-1, 2025-2 동일) |

### 데이터 교환 방식 ★★ (2023-1, 2024-2 동일)
- **가상 회선 방식**: 연결형, 경로를 미리 **논리적으로 고정**
- **데이터그램 방식**: 비연결형, 접속 절차 없이 헤더 정보로 개별 전달
- (회선 교환은 물리적 경로 고정, 패킷 교환 = 가상회선 + 데이터그램)

### IP 주소 계산 ★★★ (매년 유형 반복)
- 서브넷 마스크로 네트워크 주소 = IP AND 마스크
- /26 (255.255.255.192): 블록 64 → 호스트 수 = 64 − 2 = **62**
- /22 (255.255.252.0): 192.168.35.100 → 네트워크 192.168.32.0 ~ 35.255
- /23 (255.255.254.0): 192.168.11.20 → **192.168.10.0/23**, 192.168.12.200 → 192.168.12.0/23 (2026-1, CIDR 표기로 답)
- FLSM n개 분할: 필요 비트 올림 (3개 → 2비트 → /26, 두 번째 브로드캐스트 192.168.1.127)
- URL 구조 ★: `scheme://authority/path?query#fragment`

### HDLC ★★ (2023-2, 2026-1 재출제)
- 프레임: **정보(I, '0')** / **감독(S, '10', 오류·흐름 제어)** / **비번호(U, '11', 모드 설정)**
- 모드: 표준 응답(NRM, 주국 허가 필요) / **비동기 균형(ABM**, 혼합국 자유 전송) / **비동기 응답(ARM**, 종국 자유 송신)

### 오류 제어 ★★ (2023-2, 2025-3 동일 문제)
- **FEC**(전진 오류 수정): 수신측이 **스스로 수정**, 재전송 불필요 → **해밍 코드**(2bit 검출, 1bit 교정)
- **BEC**(후진 오류 수정): 오류 시 **재전송 요구**(ARQ) → **패리티 검사**(1bit 검출), **CRC**(다항식, HDLC FCS, 집단 오류 검출) ★
- 기타: Ad-hoc Network ★(고정 유선망 없이 모바일 호스트만, 멀티 홉 라우팅, 재난·군사)

## 9. 2020~2022 선별 이론 (용어 풀 확장)

> 2020~2021년은 용어 단답 비중이 컸던 시기. 아래 용어들이 재출제 풀에 계속 남아 있음.

### 데이터베이스 심화
- **트랜잭션 ACID**: 원자성(모두 반영 or 전혀 반영 X) / 일관성 / 독립성(격리성) / 지속성
- **ROLLBACK**: 오류 시 SAVEPOINT 시점으로 복귀 / **REDO**: 로그로 재실행, **UNDO**: 작업 취소
- **회복 기법**: 지연 갱신(로그만 기록) / **즉각 갱신**(Redo+Undo 모두 사용) / 체크 포인트 / 그림자 페이징
- **병행제어**: **로킹**(상호배제), 타임 스탬프 순서, 낙관적 검증, 다중 버전
- **이상 현상 3가지**: 삽입 이상 / 삭제 이상 / 갱신 이상
- **함수 종속**: Full(완전) / Partial(부분 — 2NF에서 제거) / Transitive(이행 — 3NF에서 제거)
- **관계해석**: 비절차적(무엇을), 튜플·도메인 해석 ↔ **관계대수**: 절차적(어떻게)
- **키 속성**: 슈퍼키 = 유일성 / 후보키 = 유일성 + 최소성
- DCL: **GRANT**(권한 부여) / **REVOKE**(권한 회수)
- SQL: `ALTER TABLE 학생 ADD 주소 VARCHAR(20)` / `CREATE INDEX 이름 ON 테이블(열)` / `WHERE 단가 > ALL (서브쿼리)` / `LIKE '이%' ORDER BY ... DESC`
- **스키마**: DB의 구조와 제약조건 정보를 담은 기본 구조 / **데이터 모델 구성요소**: 연산·구조·제약조건

### 소프트웨어 공학·테스트 심화
- **테스트 기본 원리**: 살충제 패러독스(동일 테스트 반복 → 새 결함 발견 못 함) ★, 파레토 법칙(20% 코드에 80% 결함), 오류-부재의 궤변
- **V모델 테스트 단계**: 단위 → 통합 → 시스템 → 인수
- **테스트 오라클**: 참 / **샘플링**(특정 몇 개 입력값만) / 휴리스틱 / 일관성 검사
- **회귀(Regression) 테스트**: 수정 후 새로 유입된 오류 확인 위한 반복 시험
- **알파 테스트**(개발사 내부) / **베타 테스트**(잠재 고객, 발매 전 최종)
- **원인-효과 그래프(Cause-Effect Graph)**: 입력(원인)-출력(결과)의 논리적 관계로 테스트케이스 도출
- **정적 분석**(실행 없이 코드 분석) / **동적 분석**(실행하며 메모리 누수 등 추적), **JUnit**(자바 단위 테스트 프레임워크)
- **리팩토링**: 외부 동작 변경 없이 단순화·가독성 ↑ / **형상 통제**: 변경 여부·활동 통제 (식별→통제→감사→기록)
- **릴리스 노트 헤더**: 문서명·제품명·버전·릴리스 날짜 등
- **LoC 계산**: 기간 = (총 라인 ÷ 1인 월 생산성) ÷ 인원 (30,000÷300÷5 = 20개월)
- **Fan-in**(들어오는 화살표) / **Fan-out**(나가는 화살표) — 그림에서 세는 문제
- **럼바우 분석**: Object(객체, ERD) → Dynamic(동적, 상태도) → Function(기능, DFD)
- **SOLID 중 ISP**: 클라이언트가 사용하지 않는 메서드에 의존하지 않도록 인터페이스 분리
- **UI 설계 원칙**: 직관성·유효성·학습성·유연성 / **UX**(경험) vs **UI**(인터페이스) / **GUI** / **NUI**(자연스러운 표현으로 제어)
- **애자일**: 사람 중심, 변화에 유연한 경량 방법론

### 네트워크·보안 심화
- **프로토콜 3요소**: 구문(Syntax) / 의미(Semantics) / 순서(Timing)
- **OSI 7계층 역할**: 물리(비트) → 데이터링크(MAC·오류/흐름) → 네트워크(경로) → 전송 → 세션 → 표현(압축·암호화) → 응용
- **IPv6**: 128비트 (IPv4는 32비트, 8비트×4)
- **IGP**(AS 내부: RIP·OSPF) / **EGP**(AS 간) / **BGP**(AS 간 정보 교환 EGP)
- **AAA**: Authentication(인증) / Authorization(인가) / Accounting(계정 관리)
- **랜드 어택**: 출발지 = 목적지 주소로 위조 / **스니핑**: 패킷 수집·재조합(수동적 공격) / **ARP 스푸핑**: ARP 메시지로 근거리 중간자 공격
- **SQL Injection**: 악의적 SQL 삽입·실행 공격
- 암호: **DES**(64비트 블록, 실질 56비트 키) / **IDEA**(64블록·128키) / **SKIPJACK**(Clipper 칩) / **MD5**(128비트 해시, RFC 1321)
- **TKIP**: 임시 키 무결성 프로토콜 / **RTO**: 업무 중단~복구까지 목표 시간 (RPO: 데이터 손실 허용 시점)
- 보안 3요소: 기밀성 / 무결성 / **가용성**(인가 사용자의 지속 사용 보장)

### 기타 용어
- **XML**(W3C, SGML 복잡함 해결 마크업) / **JSON**(속성-값 쌍, AJAX에서 XML 대체) / **SOAP**(XML 기반, RESTful로 대체 가능) / **WSDL**(웹 서비스 기술 XML 언어)
- **HTTP / Hypertext / HTML** 구분 문제
- **EAI**: 기업 애플리케이션 통합 — 유형: Point-to-Point / Hub & Spoke / Message Bus / Hybrid ★
- **IPC**: 프로세스 간 통신 (공유 메모리, 소켓, 세마포어, 메시지 큐)
- **HRN 우선순위** = (대기시간 + 서비스시간) ÷ 서비스시간
- **프로세스 상태**: 준비(Ready) → 실행(Run) → 대기(Wait)
- **RAID 0**: 스트라이핑(분산 저장, 빠름) / RAID 1: 미러링
- **데이터 마이닝** / **블록체인** / **하둡**(분산 컴퓨팅, 자바 프레임워크) / **LOD**(Linked Open Data) / **안드로이드** / **유닉스**(C 구현, 이식성, 트리 구조)
- **헝가리안 표기법**: 접두어에 자료형 (카멜 goodMan / 파스칼 GoodMan / 스네이크 good_man)
- **Factory Method**: 하위 클래스가 생성할 객체 결정 / 패턴 목적 분류: 생성·구조·**행위**
- UML 클래스 관계 영문: Association / **Aggregation**(집약, 전체-부분) / Composition / **Generalization**(일반화, IS-A) / Dependency / Realization

## 답안 작성 주의사항 (감점 방지)

1. **대소문자 구분**: `BACDE`를 `bacde`로 쓰면 오답
2. **출력 형식 준수**: 줄바꿈으로 출력되는 것을 쉼표로 나열하면 오답 (예: `10, 11, 10, 20` ✗)
3. 출력값 사이 공백·콤마 임의 추가 금지
4. 영문 약어 요구 시 약어로, 3글자 지정 시 3글자로 (SSH, VPN, OTP 등)
5. 보기에서 고르라고 하면 **보기의 표기 그대로** (기호로 쓰라면 기호로)
6. SQL은 세미콜론 생략 가능, 문자열은 작은따옴표
